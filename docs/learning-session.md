## 🧘 Break Guidelines

### Kapan Harus Break
- ⏸️ **Setelah 1-2 fitur selesai** (60-90 menit kerja)
- ⏸️ **Saat merasa focus berkurang** (perhatian sulit fokus ke code)
- ⏸️ **Saat stuck > 10 menit** pada satu error (fresh mind lebih baik)
- ⏸️ **Akhir setiap jam** (5-10 menit mini break)

### Indikator Istirahat Diperlukan
❌ **Tanda-tanda butuh break:**
- Mulai membaca code berulang kali
- Sulit membuat keputusan simple (mau bikin feature apa)
- Sering typo atau logic error
- Merasa frustrated dengan test
- Mata terasa lelah / nyeri

✅ **After break indicators (selesai istirahat):**
- Merasa segar dan siap lanjut
- Focus kembali normal (tidak fluktuatif)
- Bisa baca error message dengan jelas
- Energy untuk problem-solving kembali

## 📋 Logging Aksi Istirahat

### Cara Mencatat
Saat istirahat, bisa update `learning-session.md` dengan format:

```markdown
## Break #1 (HH:MM - HH:MM)
- **Durasi:** 15 menit
- **Activity:** Makan snack + stretch
- **Reason:** Setelah selesai Habit Tracker test
- **Feeling Before:** Merasa fokus bagus, tapi mata mulai pegal
- **Feeling After:** Segar, siap lanjut Pomodoro

## Resume Coding
- **Waktu:** HH:MM
- **Fitur Next:** Pomodoro Timer (Issue #3)
```

### Auto-Log dengan Pomodoro
Aplikasi Pomodoro kita sendiri bisa catat:
- Sesi fokus berapa banyak = proxy untuk durasi belajar
- Berapa break yang diambil
- Habits yang selesai per session
